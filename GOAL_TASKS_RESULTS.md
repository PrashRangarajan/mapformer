# Goal-directed task suite — sequential, switching, noisy

Three behavioural tests of the cognitive map beyond single point-to-point navigation:
- **multistop**: announce two goals up front, navigate A then B (sequential planning).
- **switching**: start toward A, mid-episode the goal switches to B (flexible replan).
- **noisy**: closed-loop BFS oracle, but execution is stochastic (p_transition=0.10).

All run on lm200, single seed, 50 epochs, eval on held-out env (obs_map seed=1000).

## Interpretation

- **multistop**: longer chained BFS path means more drift between subgoals.
  Predicted: correction's advantage GROWS with #stops vs Vanilla.

- **switching**: the switch-token interrupts the navigate stream.
  Predicted: all correction variants similar; Vanilla worst because
  drift on the pre-switch path leaves its map further off.

- **noisy**: closed-loop BFS oracle replans after each (stochastic)
  executed action. Predicted: Level15 ≫ Vanilla — correction matters
  most when each commanded action might not execute.
