# Vector navigation probe v2 — proper Tolman cognitive-map test

Following Banino 2018 / Whittington 2020 / Tolman 1948:

- Take a prediction-trained model + a random-walk trajectory
- For each pair of timesteps (t, s), agent was at cells pos_t and pos_s
- Linear probe: (h_t, h_s) → BFS-optimal next action from pos_t to pos_s
- The probe must compute Δposition from two encoded states. Only works if
  the model's representation encodes position cleanly.

**Shortcut filter:** a (t, s) pair is 'shortcut' if BFS distance ≥ 3 AND
|t-s| ≥ 3 × bfs_distance — the agent took a long detour between visiting
the two cells. Predicting direction for shortcut pairs tests TRUE spatial
reasoning (the agent never traversed pos_t → pos_s directly).

Multi-seed (n=3 prediction-trained lm200 checkpoints). Held-out env.

## Interpretation

- RoPE near chance: standard transformer has no position encoding for vector nav
- MapFormer family well above chance: cognitive map supports linear extraction of relative position
- Shortcut accuracy ≈ non-shortcut accuracy: TRUE vector navigation. The model
  predicts directions correctly even for cell-pairs it has never traversed directly.
- Shortcut ≪ non-shortcut: the cognitive map is more like a 'trajectory memory'
  than a true spatial map.
