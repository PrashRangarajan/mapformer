## The DoG kernel bug (important)

Earlier `model_level15_dog.py` and the first version of `continuous_nav.py`
computed DoG targets as:

```python
gE = exp(-d² / (2 σE²))     # at d=0: gE = 1
gI = exp(-d² / (2 σI²))     # at d=0: gI = 1
target = max(0, gE - gI)    # at d=0: target = 0  ← BUG
```

Both Gaussians equal 1 at the centre; their difference is 0; ReLU gives 0
everywhere. **The DoG target was silently all zeros**. The earlier
`DOG_RESULTS.md` (max grid score 0.036, 0% units > 0.3) was on broken
targets — a vacuous negative result, not a real test of Sorscher's
conditions.

Fix uses *normalised* 2D Gaussians (1/σ² prefactor) so the narrower peak
is taller:

```python
gE = (1/σE²) · exp(-d² / (2 σE²))   # at d=0: 1/σE² = 0.444
gI = (1/σI²) · exp(-d² / (2 σI²))   # at d=0: 1/σI² = 0.111
target = max(0, gE - gI)            # at d=0: 0.333  ✓
```

Both files patched. Re-running the discrete Level15_DoG hex test
(`DOG_RESULTS_FIXED.md`) once GPUs free up.

## Reframings to keep using going forward

### Action noise = stochastic-transition MDP

For uniform policies, action-token corruption (post-hoc) and stochastic
execution (env rolls a coin) produce identical (action_record, observation)
data distributions. From the model's POV both look like "the next state
doesn't always match what my recorded action predicts". Use the
stochastic-transition framing — it's standard control/RL vocabulary.

`environment.py` now supports both via `p_action_noise` (post-hoc record
corruption) and `p_transition_noise` (execution-time stochasticity). The
queued pipeline runs both and compares.

### NLL is the more interesting metric than accuracy

Two models can have identical accuracy but very different NLL — one may be
confidently wrong on its mistakes, the other hedges. NLL captures
*calibration*. Most cognitive-map work doesn't report it; we do, and
Level 1.5 dominates the metric (often 2× lower) even when accuracy is
tied.

Concrete: MiniGrid clean OOD T=2048: Vanilla 0.823/0.547, Level15 0.833/0.406
— ~1pp accuracy, 26% better NLL.

### MapFormer ↔ Jane Street group-theory blog

The post (https://blog.janestreet.com/using-group-theory-to-explore-positional-encodings-attention/)
characterises positional encodings as one-parameter Lie group representations
A(τ) = exp(τG). RoPE makes τ a function of token index t. **MapFormer makes
τ a function of cumulative content** — same exp(τG) form, same SO(2)^B
group, but breaks the post's "translation-invariance in token index"
assumption to instead have translation-invariance in path-integrated state.
Complementary, not contradictory; the post stays in one corner (token-indexed),
MapFormer occupies a different corner (content-derived τ).

The post's stated open territory — multi-parameter SE(2)/SE(3) groups —
is exactly the natural extension our continuous-nav work is taking.

### Level 1.5's wrap as Lie-group-natural correction

The wrap on innovations `atan2(sin(z - θ̂), cos(z - θ̂))` is the **geodesic
on SO(2)**, not a hack. It's required because SO(2) is compact (unlike Z
which is what RoPE sits on, and unbounded), so corrections must respect
the group's exponential map. Markovic et al. 2017 derive this for SO(2)
specifically. Cite this when defending against "isn't this just a
hand-tuned trick".

## Categorising real navigation noise (for paper-defense use)

| # | Type | What it corrupts | Real examples | Right tool |
|---|---|---|---|---|
| 1 | Process / motion | True state evolution | Wheel slip, wind, gait variability | KF (and our InEKF) |
| 2 | Proprioceptive / action-record | Recorded action; execution was correct | IMU drift, vestibular noise, packet loss | KF (our action-noise test models this) |
| 3 | Continuous obs noise | Continuous sensor reading | Camera pixel noise, LiDAR jitter, GPS in clear sky | KF with R_t (Level 1.5 already does this) |
| 4 | Obs outliers | Sensor faults | Mirror reflections, GPS multipath, occlusion | Robust KF, particle filter |
| 5 | Aliasing / ambiguity | Multiple states give same obs | Hallway problem, featureless terrain | Particle filter, HMM, attention |
| 6 | Calibration drift | Sensor model parameters | Wrong wheel diameter, magnetic compass | Augmented EKF |
| 7 | Map / loop-closure | Stored map vs world | SLAM perceptual aliasing | Pose-graph, factor graphs |

Level 1.5 is the right tool for 1–3. **Attention** (not Kalman) handles 5.
We do *not* claim Level 1.5 generalises to 4, 6, 7.
